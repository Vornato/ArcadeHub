# ლევანს ბოქსი

`ლევანს ბოქსი` არის ბრაუზერზე გაშვებადი ქართული party-game პაკეტი ერთი საერთო ეკრანისთვის და ტელეფონიდან შეერთებული მოთამაშეებისთვის.

პაკეტში შედის 8 თამაში:
- `ცრუ წარწერა`
- `ტყუილის დეტექტივი`
- `პანიკური პიჩი`
- `მემის სასამართლო`
- `საიდუმლო საბოტაჟი`
- `ცუდი მთარგმნელი`
- `დახატე და მოატყუე`
- `გადარჩი სცენარს`

## შესაძლებლობები

- სრულად ქართული UI ჰოსტისთვისაც და ტელეფონის კონტროლერებისთვისაც
- `2-8` მოთამაშე
- `8` მოთამაშის შემდეგ აუდიტორიის რეჟიმი, თუ ჩართულია
- LAN / ერთ Wi‑Fi ქსელში შეერთება
- QR კოდი და ოთახის მოკლე კოდი
- Socket.IO რეალურ დროში სინქრონიზაცია
- თითოეული თამაშისთვის ცალკე მოდული
- JSON კონტენტი ყველა რეჟიმისთვის

## ლოკალური გაშვება

### 1. შედი საქაღალდეში

```powershell
cd "Games\Levans Box"
```

### 2. დააყენე პაკეტები

```powershell
npm install
```

### 3. სურვილის შემთხვევაში თავიდან გენერაცია გაუკეთე კონტენტს

```powershell
npm run generate:data
```

### 4. გაუშვი სერვერი

```powershell
npm start
```

ან Windows-ზე პირდაპირ გაუშვი:

```text
Start Levans Box Server.bat
```

### 5. გახსენი თამაში

- ჰოსტის ეკრანი: `http://127.0.0.1:3094/`
- ტელეფონები: `http://შენი-LAN-IP:3094/play`

სერვერი გაშვებისას კონსოლში თვითონაც დაბეჭდავს LAN მისამართს.

## როგორ ერთდებიან ტელეფონები

1. ჰოსტი ხსნის მთავარ ეკრანს.
2. ჰოსტი ქმნის ოთახს.
3. ეკრანზე ჩნდება ოთახის კოდი, მისამართი და QR კოდი.
4. მოთამაშეები თავიანთ ტელეფონებზე ხსნიან `/play` გვერდს.
5. შეჰყავთ ოთახის კოდი და მეტსახელი.
6. მერვე მოთამაშის შემდეგ შემომსვლელები გადადიან აუდიტორიაში, თუ აუდიტორია ჩართულია.

## სასარგებლო ბრძანებები

```powershell
npm start
npm run check
npm run generate:data
npm run smoke
```

## smoke test

`npm run smoke` ამოწმებს:

- ჰოსტისა და ტელეფონის გვერდების გახსნას
- bootstrap/API პასუხს
- 8 თამაშის launch-ს
- `8` მოთამაშე + `1` აუდიტორია შეერთებას
- ტექსტის, ხმის მიცემისა და drawing ფაზებს
- საბოლოო `final` ფაზამდე მისვლას

## შენიშვნები

- moderation toggle ჩართული რომ იყოს, ტექსტური პასუხები მარტივ ფილტრზე გადის.
- თუ კავშირი დროებით გაწყდა, მოთამაშეს შეუძლია იმავე სესიით დაბრუნება.
- ჰოსტი აკონტროლებს დაწყებას, pause-ს, შემდეგ ფაზას, match end-ს და lobby reset-ს.
